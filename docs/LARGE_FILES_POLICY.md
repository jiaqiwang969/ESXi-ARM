# 大文件管理策略

为了保证仓库可同步、可审阅、可复现，本项目采用“脚本 + 文档”优先，不直接提交大体积二进制。

## 默认不提交的内容

- 安装介质与发行包：`*.iso`, `*.zip`, `*.deb`, `*.vib`
- 虚拟机与固件产物：`*.qcow2`, `*.fd`, `*.img`
- 解包后的 payload 与实验输出：`work/out/**`, `work/iso/**`, `work/analysis/**`, `work/firmware/**`, `work/vm/**`
- 大量原始日志：`*.log`

## 必须提交的关键内容

- 可复现脚本：`work/scripts/*.sh`, `work/scripts/*.py`
- 进展与决策文档：`docs/*.md`, `README.md`

## “关键大文件”处理原则

如果某个大文件对问题复现不可替代：

1. 先在文档说明“为什么必要”；
2. 优先提交提炼后的关键片段（而不是全部原始文件）；
3. 必要时再单独上传完整文件（建议走发行页、对象存储或 LFS）。
